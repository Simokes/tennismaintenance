# Contrôle d’accès du site

## Statut

À compléter après relevé de la marque, du modèle et du type de groom.

## Fonctionnement général

Les accès utilisent un clavier à code, une ventouse électromagnétique et un groom assurant la fermeture automatique.

```text
clavier à code → libération de la ventouse → ouverture
                                      ↓
                         groom → fermeture automatique
```

## Changer un code — méthode générique

1. Vérifier l’autorisation de modification.
2. Consulter la notice officielle du modèle installé.
3. Passer le clavier en mode programmation selon cette notice.
4. Ajouter, modifier ou supprimer le code utilisateur concerné.
5. Quitter le mode programmation.
6. Vérifier que l’ancien code est refusé si nécessaire.
7. Tester le nouveau code.
8. Vérifier la libération de la ventouse.
9. Vérifier la fermeture complète par le groom.
10. Vérifier le verrouillage final.
11. Mettre à jour le registre papier conservé au bureau.
12. Tracer seulement la date, l’accès et le motif dans Git.

!!! danger "Aucun secret dans Git"
    Ne jamais inscrire dans le dépôt un code utilisateur, un code maître, une séquence secrète de programmation ou une photo les révélant.

## Contrôles après intervention

- ouverture correcte ;
- ancien code refusé lorsqu’il a été remplacé ;
- fermeture complète et régulière ;
- verrouillage effectif ;
- absence de frottement ou de claquement anormal ;
- autres accès non affectés si la programmation est commune.
